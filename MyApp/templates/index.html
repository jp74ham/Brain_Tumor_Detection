Schema and ER diagram

Consists of five key entities

  • Patients table stores anonymized patient data including demographics and scan details
  • MRI_Scans table holds metadata for each MRI image linked to patients
  • Tumor_Classification table contains model predictions with tumor types and confidence scores
  • Model_Performance table tracks accuracy metrics over time
  • User_Roles table manages access levels for different users.

Relationships

  • One Patient can have multiple MRI_Scans (1-to-Many)
  • Each MRI_Scan has one Tumor_Classification (1-to-1)
  • Multiple Tumor_Classification can reference a single Model_Performance record (Many-to-1)
  • A User_Role can be associated with multiple MRI uploads (1-to-Many)

ETL Pipeline
  • Extract: Downloaded brain MRI images from Kaggle dataset containing 4 tumor types (glioma, meningioma, pituitary, no_tumor)
  • Transform: Preprocessed images - resized to 224x224, converted to RGB, normalized pixel values, and extracted metadata
  • Load: Populated SQLite database with 5 tables - Patients (500 synthetic records using Faker), MRI_Scans, Tumor_Classification, Model_Performance, and User_Roles
  • Created relational links between patients, scans, and classifications with proper foreign key constraints
  • Generated ground truth labels from dataset folder structure for model training and validation

Query 1: Total cases by tumor type

SELECT label AS tumor_type,
       COUNT(*) AS total_cases
FROM mri_scans
GROUP BY label
ORDER BY total_cases DESC;

Query 2: Age group analysis with average image area

SELECT
    CASE
        WHEN age BETWEEN 20 AND 29 THEN '20s'
        WHEN age BETWEEN 30 AND 39 THEN '30s'
        WHEN age BETWEEN 40 AND 49 THEN '40s'
        WHEN age BETWEEN 50 AND 59 THEN '50s'
        WHEN age BETWEEN 60 AND 69 THEN '60s'
        ELSE '70s+'
    END AS age_group,
    label,
    AVG(orig_width * orig_height) AS avg_area,
    COUNT(*) AS cases
FROM mri_scans
GROUP BY age_group, label
ORDER BY age_group;

Query 6: Weekly case trends by tumor type

SELECT
    STRFTIME('%Y-%W', scan_date) AS week,
    label AS tumor_type,
    COUNT(*) AS cases
FROM mri_scans
GROUP BY week, tumor_type
ORDER BY week;

Query 7: MRI scans for specific tumor types with patient info

SELECT
    ms.scan_id,
    p.anonymized_id,
    p.age,
    p.gender,
    ms.image_path,
    tc.tumor_type,
    tc.confidence_score,
    p.hospital_name
FROM MRI_Scans ms
JOIN Patients p ON ms.patient_id = p.patient_id
JOIN Tumor_Classification tc ON ms.scan_id = tc.scan_id
WHERE tc.tumor_type = 'glioma'
ORDER BY tc.confidence_score DESC
LIMIT 10;

Query 8: Hospital performance statistics

SELECT
    p.hospital_name,
    COUNT(DISTINCT p.patient_id) as total_patients,
    COUNT(ms.scan_id) as total_scans,
    SUM(CASE WHEN tc.tumor_type != 'no_tumor' 
        THEN 1 ELSE 0 END) as tumor_detections,
    ROUND(SUM(CASE WHEN tc.tumor_type != 'no_tumor' 
        THEN 1 ELSE 0 END) * 100.0 /
        COUNT(ms.scan_id), 2) as detection_rate
FROM Patients p
JOIN MRI_Scans ms ON p.patient_id = ms.patient_id
JOIN Tumor_Classification tc ON ms.scan_id = tc.scan_id
GROUP BY p.hospital_name
ORDER BY total_scans DESC;

Query Result

Query 9: Most recent scans by tumor type

SELECT
    tc.tumor_type,
    COUNT(*) as recent_count,
    MAX(ms.upload_timestamp) as latest_scan
FROM Tumor_Classification tc
JOIN MRI_Scans ms ON tc.scan_id = ms.scan_id
WHERE ms.upload_timestamp >= date('now', '-30 days')
GROUP BY tc.tumor_type
ORDER BY recent_count DESC;
